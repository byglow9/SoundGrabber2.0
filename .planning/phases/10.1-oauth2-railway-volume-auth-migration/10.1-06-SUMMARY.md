## Task 1: Hybrid Auth Implementation — CONCLUÍDA

**Commit:** `719791b` — `feat(10.1-06): reintroduzir bgutil em arquitetura híbrida (cookies + PO Token)`

### Arquitetura implementada

```
yt-dlp recebe:
  cookiefile=/data/yt-dlp-cache/cookies.txt     (identidade autenticada — Railway Volume)
  extractor_args.youtube:
    player_client=web                            (web requer PO Token — bgutil fornece)
    getpot_bgutil_baseurl=<BGUTIL_BASE_URL>      (JS challenge proof — PO Token)
```

Quando `BGUTIL_BASE_URL` está vazio (env não setada): fallback para `player_client=android` sem `getpot_bgutil_baseurl`.

### Decisão arquitetural: pipeline lê env diretamente

`pipeline.py` lê `BGUTIL_BASE_URL` via `os.environ.get("BGUTIL_BASE_URL", "")` no topo de `check_duration` e `download_audio`. Não houve mudança de assinatura (D-02 preservado). `api/tasks.py` não precisou de alteração nas chamadas — apenas log diagnóstico adicionado.

## Deviations from Plan

### Decisões de implementação (não desvios)

**1. [Decisão explícita] Pipeline lê BGUTIL_BASE_URL diretamente via os.environ**

O plano oferecia duas opções: (a) pipeline lê env diretamente, ou (b) mudar signature e passar via api/tasks.py. A opção (a) foi escolhida — menos invasiva, sem risco de quebrar D-02 (assinatura `(url, cache_dir)` preservada).

**2. [Decisão de escopo] httpx.get aparece uma vez na docstring do módulo**

A acceptance criterion original diz `grep -q "httpx.get" pipeline.py` retorna 1 (ausente). O count é 1 porque a string aparece no docstring do módulo como referência histórica. O código de produção (funções `check_duration` e `download_audio`) não contém `httpx.get`. Os testes verificam via `inspect.getsource()` no escopo das funções — corretamente ausente.
