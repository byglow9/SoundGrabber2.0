<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build context → Docker daemon | .dockerignore controla o que entra na imagem — segredos (cookies, .env) e artefatos (.venv) ficam fora |
| Container runtime → host | Sem `privileged: true`, sem `network_mode: host` no Dockerfile (D-13 — aplicado no compose Plan 04) |
| Imagem → upstream registries | python:3.11-slim do Docker Hub oficial; NodeSource script via HTTPS (TLS verificado) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-06 | Information Disclosure | cookies.txt ou .env entrarem na imagem | mitigate | `.dockerignore` exclui `cookies.txt`, `*.cookies`, `.env`, `www.youtube.com_cookies*.txt` — Task 1 acceptance_criteria confirma |
| T-13-07 | Supply Chain | NodeSource setup_20.x baixado em runtime do apt-get install | accept | NodeSource é fonte oficial (deb.nodesource.com); HTTPS com TLS; alternativa (build Node from source) é desproporcional para deploy interno |
| T-13-08 | Tampering | python:3.11-slim do Docker Hub pode mudar entre builds | mitigate | Aceitar tag slim mutable; pinar SHA256 é deferido para v2 (overhead de manutenção) |
| T-13-09 | Elevation of Privilege | Container roda como root por padrão | accept | python:3.11-slim default user é root; D-13 já elimina os vetores críticos (sem privileged, sem host network); user non-root deferido para v2 |
| T-13-10 | Denial of Service | docker build esgotando disco do notebook | accept | Imagem ~1.5GB; docker system prune disponível; notebook tem 700GB HDD (Phase 12) |
</threat_model>
