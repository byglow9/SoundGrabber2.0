    ```python
    @app.get("/jobs/{job_id}")
    def get_job(job_id: str) -> dict:
        # Defensive: reject malformed job_ids (path traversal, injection, oversized inputs).
        if not JOB_ID_PATTERN.match(job_id):
            raise HTTPException(status_code=404, detail="Job not found")

        # Pitfall 1 / D-02: PENDING is ambiguous between "in queue" and "never existed".
        # The Redis Set populated by POST /jobs is the authoritative registry.
        if not _redis.sismember(JOB_REGISTRY_KEY, job_id):
            raise HTTPException(status_code=404, detail="Job not found")

        result = AsyncResult(job_id, app=celery_app)
        state = result.state

        if state in ("PENDING", "STARTED"):
            return {"status": "queued"}

        if state == "DOWNLOADING":
            meta = result.info if isinstance(result.info, dict) else {}
            return {"status": "downloading", "stage": meta.get("stage")}

        if state == "CONVERTING":
            meta = result.info if isinstance(result.info, dict) else {}
            return {"status": "converting", "stage": meta.get("stage")}

        if state == "ANALYZING":
            meta = result.info if isinstance(result.info, dict) else {}
            return {"status": "analyzing", "stage": meta.get("stage")}

        if state == "SUCCESS":
            data = result.result if isinstance(result.result, dict) else {}
            # D-05: wav_path is internal — strip it before responding.
            return {
                "status": "done",
                "bpm": data.get("bpm"),
                "bpm_half": data.get("bpm_half"),
                "bpm_double": data.get("bpm_double"),
                "key": data.get("key"),
                "camelot": data.get("camelot"),
                "duration_sec": data.get("duration_sec"),
                "download_url": data.get("download_url"),
            }

        if state == "FAILURE":
            # Pitfall 3: result.result is the exception object in FAILURE state.
            # JobFailure carries .error and .error_type; non-JobFailure exceptions fall back.
            exc = result.result
            error = getattr(exc, "error", None)
            error_type = getattr(exc, "error_type", None)
            if error is None or error_type is None:
                # Defensive: a non-JobFailure exception escaped; sanitize.
                error = "An internal error occurred. Please try again."
                error_type = "internal_error"
            return {"status": "failed", "error": error, "error_type": error_type}

        # Unknown Celery state — treat as queued (will resolve next poll).
        return {"status": "queued"}
    ```

    ```python
    def sweep_expired_wavs(directory: Path, ttl_seconds: int) -> int:
        """Delete sg_*.wav files in `directory` older than `ttl_seconds`. Returns count deleted.

        Pure function — testable without threads. The daemon loop calls this every 60 seconds.
        D-01: TTL matches the 15-minute WAV lifecycle decision.
        """
        deleted = 0
        now = time.time()
        for wav in Path(directory).glob("sg_*.wav"):
            try:
                if now - wav.stat().st_mtime > ttl_seconds:
                    wav.unlink(missing_ok=True)
                    deleted += 1
            except OSError:
                # File vanished mid-sweep or permissions issue — skip.
                continue
        return deleted


    def _run_sweeper_loop() -> None:
        """Daemon thread: sweep /tmp for expired sg_*.wav files every 60 seconds.
        Survives individual sweep errors — thread MUST NOT die on OSError."""
        while True:
            try:
                count = sweep_expired_wavs(Path("/tmp"), settings.wav_ttl)
                if count > 0:
                    logger.info("wav-sweeper: deleted %d expired WAV file(s)", count)
            except Exception:  # noqa: BLE001 — sweeper must never crash
                logger.exception("wav-sweeper iteration failed; continuing")
            time.sleep(60)
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> GET /jobs/{job_id} | Untrusted job_id in URL path parameter |
| client -> GET /files/{job_id} | Untrusted job_id in URL path; success path serves a file from /tmp filesystem |
| Celery result dict -> download_file | Worker-controlled wav_path field used to construct FileResponse — must be validated |
| /tmp filesystem -> sweeper | Daemon thread reads and deletes files from /tmp |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Tampering / Information Disclosure | job_id path parameter in GET /jobs and GET /files | mitigate | `JOB_ID_PATTERN = ^[a-zA-Z0-9-]{1,64}$` regex guard on both GET endpoints — rejects ../path, %2F encoded slashes, null bytes, overlong strings. Celery UUIDs (36 chars, hex + hyphens) always pass this pattern. |
| T-02-03-02 | Information Disclosure | wav_path returned to API consumer via GET /jobs | mitigate | get_job SUCCESS branch hard-codes the response key list and excludes wav_path. Acceptance criteria greps confirm the omission. |
| T-02-03-03 | Information Disclosure | Internal exception text in FAILURE response | mitigate | Only JobFailure.error (sanitized hard-coded string) and JobFailure.error_type (enum) are returned. Fallback for non-JobFailure exceptions returns canonical "internal_error" message. |
| T-02-03-04 | Tampering / Arbitrary File Read | wav_path injected outside /tmp via compromised worker | mitigate | download_file resolves wav_path and asserts relative_to(/tmp) AND name.startswith("sg_"). Two independent checks. If either fails: 410, not the file. Acceptance criteria verify both checks are present. |
| T-02-03-05 | Denial of Service | /tmp disk fill from accumulated WAVs | mitigate | sweep_expired_wavs runs every 60s deleting files older than 900s. Steady-state bound: ~(3 jobs/min rate limit in Phase 3 * 15 min TTL * ~30MB WAV) = ~1.4 GB peak before Phase 3 rate limiting. Sweeper prevents unbounded accumulation. |
| T-02-03-06 | Denial of Service | Sweeper thread crash stopping cleanup | mitigate | _run_sweeper_loop wraps the iteration body in try/except Exception; logs and continues. Thread is daemon so process death is the ultimate cleanup. Acceptance criterion verifies the exception wrapper exists. |
| T-02-03-07 | Information Disclosure | Job ID enumeration | accept | Celery task IDs are uuid4 (122 random bits). Brute-force guessing is computationally infeasible. Phase 3 adds rate limiting that further limits probe attempts. |
| T-02-03-08 | Spoofing | HTTP Range requests bypassing access control | accept | FileResponse honors Range headers (enables browser progressive download). Access control (state==SUCCESS check) runs before FileResponse is constructed, so Range only applies post-authorization. No bypass is possible. |
