<objective>
Close the loop on Phase 2's HTTP contract by implementing the polling and download endpoints, plus the lifecycle that keeps the system clean.

This plan replaces the two remaining 501 stubs in `api/main.py`:
1. `GET /jobs/{job_id}` — reads `AsyncResult` from Celery's Redis backend, maps Celery state names to the 6-value API contract (queued/downloading/converting/analyzing/done/failed), unpacks JobFailure into the {error, error_type} shape per D-05/D-06, and returns 404 for unknown/expired job_ids by checking the `sg:jobs` Redis Set first (Pitfall 1, D-02).
2. `GET /files/{job_id}` — verifies the job succeeded, then streams the WAV via `FileResponse` (no `read()` into memory — Pattern 4). Returns 404 if not yet done, 410 if the WAV expired off disk.

Plus the ambient infrastructure:
- `sweep_expired_wavs(directory, ttl_seconds)` helper — pure, testable, deletes `sg_*.wav` files older than ttl
- `_run_sweeper_loop()` — daemon thread loop that calls the helper every 60 seconds against `/tmp` and `settings.wav_ttl`
- Lifespan launches the thread at startup; daemon flag means it dies with the process (D-07 dev workflow — no separate Beat process)

The plan ends with a `checkpoint:human-verify` task that exercises the full 3-concurrent-curl scenario from ROADMAP success criteria #4. This is genuinely manual: it requires Redis + worker + uvicorn all running concurrently with real (or live-mocked) jobs, which an `automated` test cannot simulate at the same fidelity.

    ```python
    def get_job(job_id: str) -> dict:
        # Defensive: job_id must look like a Celery UUID (no path traversal, no injection).
        if not JOB_ID_PATTERN.match(job_id):
            raise HTTPException(status_code=404, detail="Job not found")

        # Pitfall 1 / D-02: PENDING is ambiguous between "in queue" and "never existed".
        # We disambiguate by checking the registry Set populated at POST /jobs.
        if not _redis.sismember(JOB_REGISTRY_KEY, job_id):
            raise HTTPException(status_code=404, detail="Job not found")

        result = AsyncResult(job_id, app=celery_app)
        state = result.state

        if state in ("PENDING", "STARTED"):
            return {"status": "queued"}

        if state == "DOWNLOADING":
            meta = result.info if isinstance(result.info, dict) else {}
            return {"status": "downloading", "stage": meta.get("stage")}

        if state == "CONVERTING":
            meta = result.info if isinstance(result.info, dict) else {}
            return {"status": "converting", "stage": meta.get("stage")}

        if state == "ANALYZING":
            meta = result.info if isinstance(result.info, dict) else {}
            return {"status": "analyzing", "stage": meta.get("stage")}

        if state == "SUCCESS":
            data = result.result or {}
            # D-05: wav_path is internal — strip before returning.
            return {
                "status": "done",
                "bpm": data.get("bpm"),
                "bpm_half": data.get("bpm_half"),
                "bpm_double": data.get("bpm_double"),
                "key": data.get("key"),
                "camelot": data.get("camelot"),
                "duration_sec": data.get("duration_sec"),
                "download_url": data.get("download_url"),
            }

        if state == "FAILURE":
            # JobFailure carries .error and .error_type per D-05/D-06.
            # Pitfall 3: result.result is the exception object in FAILURE; result.info varies.
            exc = result.result
            error = getattr(exc, "error", None)

                error = "An internal error occurred. Please try again."

            return {"status": "failed", "error": error, "error_type": error_type}

        # Unknown Celery state — treat as queued (will resolve next poll).
        return {"status": "queued"}
    ```

    ```python
    def sweep_expired_wavs(directory: Path, ttl_seconds: int) -> int:
        """Delete sg_*.wav files in `directory` older than `ttl_seconds`. Returns count deleted.

        Pure helper — testable without threads. The daemon loop calls this every 60s.
        """
        deleted = 0
        now = time.time()
        for wav in Path(directory).glob("sg_*.wav"):
            try:
                if now - wav.stat().st_mtime > ttl_seconds:
                    wav.unlink(missing_ok=True)
                    deleted += 1
            except OSError:
                # File vanished mid-sweep, permission issue, etc. — ignore.
                continue
        return deleted


    def _run_sweeper_loop() -> None:
        """Daemon thread loop: sweep /tmp every 60s. Dies with the process (daemon=True)."""
        while True:
            try:
                count = sweep_expired_wavs(Path("/tmp"), settings.wav_ttl)
                if count > 0:
                    logger.info("wav-sweeper: deleted %d expired WAV(s)", count)
            except Exception:                  # noqa: BLE001 — sweeper must NEVER die
                logger.exception("wav-sweeper iteration failed; continuing")
            time.sleep(60)
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → GET /jobs/{job_id} | Untrusted job_id passed in URL path |
| client → GET /files/{job_id} | Untrusted job_id passed in URL path; success path serves a file from disk |
| process_job result → /files/{id} | Worker-controlled `wav_path` field consumed by FileResponse — must be defended against arbitrary path |
| /tmp filesystem → sweeper | Filesystem state mutated by daemon thread |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Tampering / Information Disclosure | job_id path parameter | mitigate | `JOB_ID_PATTERN = ^[a-zA-Z0-9-]{1,64}$` regex guard on both GET endpoints — rejects path traversal attempts (`../`, `%2F`), null bytes, oversized inputs. UUIDs from Celery match this pattern; anything else returns 404 |
| T-02-03-02 | Information Disclosure | wav_path leaked to API consumer | mitigate | get_job's SUCCESS branch hard-codes the response key list and explicitly omits `wav_path`. Acceptance criterion greps for the omission |
| T-02-03-03 | Information Disclosure | Internal exception text leaked via FAILURE branch | mitigate | Only `JobFailure.error` (sanitized) and `JobFailure.error_type` (enum) are returned. Defensive fallback for non-JobFailure exceptions returns hard-coded "internal_error" message |
| T-02-03-04 | Tampering / SSRF on disk | Attacker controls wav_path → reads /etc/passwd | mitigate | download_file resolves wav_path and asserts it is `relative_to(/tmp)` AND basename starts with `sg_`. Two-layer check defends against worker-side compromise injecting `../etc/passwd` into result dict |
| T-02-03-05 | Denial of Service | /tmp disk fill | mitigate | sweep_expired_wavs runs every 60s deleting WAVs older than 900s. Worst-case storage: ~(3 jobs/min × 15 min × ~30MB WAV) ≈ 1.4 GB transient. Sweeper bounds steady state. Phase 3 will add per-IP rate limiting to bound submission rate. |
| T-02-03-06 | Denial of Service | Sweeper thread crashes silently | mitigate | `_run_sweeper_loop` wraps the per-iteration body in try/except (logs `logger.exception`, continues). Acceptance criterion verifies the wrapper exists. |
| T-02-03-07 | Information Disclosure | Job ID enumeration | accept | Celery task IDs are uuid4 (122 random bits) — guessing space is intractable. Brute-force probing is rate-limited in Phase 3. |
| T-02-03-08 | Spoofing | Range request header bypassing access control | accept | FileResponse honors Range requests by design (good — supports browser progressive download). Access control runs BEFORE FileResponse is constructed (state==SUCCESS check), so range only applies post-authorization. |
</threat_model>
