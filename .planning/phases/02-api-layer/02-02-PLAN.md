    ```python
    @celery_app.task(bind=True, name="soundgrabber.process_job")
    def process_job(self, url: str) -> dict[str, Any]:
        """Pipeline orchestration: check_duration → download_audio → analyze_audio.

        Emits custom Celery states between stages so GET /jobs/{id} can report progress.
        Maps pipeline exceptions to sanitized D-05/D-06 errors via JobFailure.
        """
        try:
            # Stage 0: duration check (CORE-05 enforced inside pipeline.check_duration)
            self.update_state(state="DOWNLOADING", meta={"stage": "checking_duration"})
            check_duration(url, settings.cookies_path)

            # Stage 1: download + convert (yt-dlp postprocessor produces WAV)
            self.update_state(state="DOWNLOADING", meta={"stage": "downloading"})
            wav_path = download_audio(url, settings.cookies_path, settings.po_token)

            self.update_state(state="CONVERTING", meta={"stage": "converting"})
            # download_audio already produced /tmp/sg_*.wav via FFmpegExtractAudio.
            # CONVERTING is reported as a discrete stage for the API contract even though
            # no extra work happens here — keeps the 5-state contract from ROADMAP intact.

            # Stage 2: analyze
            self.update_state(state="ANALYZING", meta={"stage": "analyzing"})
            result = analyze_audio(wav_path)

            return {
                "status": "done",
                "bpm": result["bpm"],
                "bpm_half": result["bpm_half"],
                "bpm_double": result["bpm_double"],
                "key": result["key"],
                "camelot": result["camelot"],
                "duration_sec": result["duration_sec"],
                "wav_path": result["wav_path"],
                "download_url": f"/files/{self.request.id}",
            }

        except ValueError as e:
            # check_duration: "Video too long" / metadata missing
            # validate_wav: "Audio invalid or corrupt" / ffprobe failures
            logger.info("Job %s validation_error: %s", self.request.id, e)
            msg_lower = str(e).lower()
            if "too long" in msg_lower or "15-minute" in msg_lower or "15 minutes" in msg_lower:
                error = "Video is too long (max 15 minutes)."
            else:
                error = "Could not validate audio file."
            raise JobFailure(error=error, error_type="validation_error") from e

        except FileNotFoundError as e:
            logger.info("Job %s download_error (file missing): %s", self.request.id, e)
            raise JobFailure(
                error="Download failed: audio file not produced.",
                error_type="download_error",
            ) from e

        except RuntimeError as e:
            # download_audio wraps yt-dlp.utils.DownloadError as RuntimeError
            logger.info("Job %s download_error: %s", self.request.id, e)
            raise JobFailure(
                error="Download failed. The video may be unavailable or blocked.",
                error_type="download_error",
            ) from e

        except Exception as e:                # noqa: BLE001 — broad catch is intentional; we sanitize
            logger.exception("Job %s internal_error", self.request.id)
            raise JobFailure(
                error="An internal error occurred. Please try again.",
                error_type="internal_error",
            ) from e
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /jobs | Untrusted JSON body containing user-supplied URL — primary attack surface |
| process_job → pipeline.py | URL passes from API into yt-dlp; yt-dlp is treated as the trust boundary for network egress |
| process_job → Celery → Redis | Task arguments serialized as JSON (pickle disabled in Plan 01) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Spoofing / Tampering | JobRequest.youtube_url | mitigate | Pydantic field_validator with strict allowlist (`{youtube.com, www.youtube.com, m.youtube.com, youtu.be}`) — verified by test_invalid_url_rejected covering `http://localhost:6379` (SSRF), `ftp://`, vimeo, soundcloud, generic example.com |
| T-02-02-02 | Server-Side Request Forgery (S/T) | yt-dlp called with user URL | mitigate | URL allowlist enforced BEFORE the URL reaches yt-dlp; yt-dlp is never invoked with non-YouTube hosts; Pydantic raises 422 at the API boundary, not deep in the worker |
| T-02-02-03 | Information Disclosure | error message in failed job response | mitigate | D-05/D-06: JobFailure exception carries hard-coded sanitized strings only; raw exception text is logged server-side via `logger.info` / `logger.exception` but never returned to the client. Acceptance criteria explicitly forbid `str(e)` substitution into JobFailure.error |
| T-02-02-04 | Information Disclosure | `wav_path` field in process_job return dict | accept | wav_path IS returned in the worker's result dict (Plan 03 needs it for FileResponse) but Plan 03's GET /jobs/{id} handler MUST strip it before responding. Documented in Plan 03 must_haves. |
