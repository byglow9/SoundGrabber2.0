## Section 1: State variables
```javascript
'use strict';
let state = 'IDLE';
let jobId = null;
let pollTimer = null;
let timeoutTimer = null;
let countdownTimer = null;
```

## Section 2: DOM refs
```javascript
const $ = id => document.getElementById(id);
```
Do NOT cache DOM refs at module level — they are not available until DOMContentLoaded. Cache inside `init()` or call `$()` inline in each function.

## Section 3: setState(newState, payload)
Central dispatcher — takes a state string and optional payload object, calls the appropriate `show*()` function.

```javascript
function setState(newState, payload = {}) {
  state = newState;
  switch (newState) {
    case 'IDLE':            showIdle(); break;
    case 'SUBMITTING':      showSubmitting(); break;
    case 'POLLING':         showPolling(payload.label || 'Processando...'); break;
    case 'DONE':            showDone(payload); break;
    case 'ERROR_VALIDATION': showErrorValidation(payload.message || ''); break;
    case 'ERROR_RATE_LIMIT': showErrorRateLimit(payload.retryAfter || 60); break;
    case 'ERROR_JOB':       showErrorJob(payload.message || 'Algo deu errado. Tente novamente.'); break;
    case 'ERROR_TIMEOUT':   showErrorTimeout(); break;
  }
}
```

## Section 4: API functions

### submitJob(url) — per D-01 through D-06 and RESEARCH.md Pattern 5
- At the start: clear ALL timers before anything else (Pitfall 4)
- POST /jobs with `{ youtube_url: url }` (NOT "url" — Pitfall 1)
- Content-Type: application/json (Pitfall 3)
- Wrap entire body in try/catch (Pitfall 5)
- Handle 202 → setState('POLLING') + startPolling(data.job_id)
- Handle 422 → determine error message from error_type:
  - if error_type === 'validation_error' and error contains 'YouTube': "URL inválida. Use um link do YouTube (youtube.com ou youtu.be)."
  - if error_type === 'validation_error' and error contains 'too long' or '15': "Vídeo com mais de 15 minutos. Escolha um vídeo mais curto."
  - else: "URL inválida. Verifique o link e tente novamente."
  - setState('ERROR_VALIDATION', { message })
- Handle 429 → `const retryAfter = parseInt(response.headers.get('retry-after') || '60', 10)` (Pitfall 6)
  - setState('ERROR_RATE_LIMIT', { retryAfter })
- Catch TypeError (network error) → setState('ERROR_JOB', { message: 'Erro de conexão. Verifique sua internet e tente novamente.' })

### startPolling(id) — per D-01, D-02
```javascript
function startPolling(id) {
  jobId = id;
  pollTimer = setInterval(pollStatus, 2000);
  timeoutTimer = setTimeout(() => {
    stopPolling();
    setState('ERROR_TIMEOUT');
  }, 180 * 1000);
}
```

### stopPolling()
```javascript
function stopPolling() {
  clearInterval(pollTimer);
  clearTimeout(timeoutTimer);
  pollTimer = null;
  timeoutTimer = null;
}
```

### pollStatus() — per D-03 and RESEARCH.md Pattern 3
- GET /jobs/${jobId} — wrap in try/catch (Pitfall 5)
- On !response.ok → stopPolling() + setState('ERROR_JOB', { message: 'Erro ao consultar status do job.' })
- On status 'done' → stopPolling() + setState('DONE', data)
- On status 'failed' → stopPolling() + determine error message:
  - error_type === 'download_error': "Não foi possível baixar o vídeo. O YouTube pode ter bloqueado o acesso. Tente novamente."
  - error_type === 'internal_error': "Erro interno. Tente novamente."
  - else: "Algo deu errado. Tente novamente."
  - setState('ERROR_JOB', { message })
- On queued/downloading/converting/analyzing → setState('POLLING', { label: stageLabel(data.status, data.stage) })

## Section 6: Helper functions

### estimateSizeMB(durationSec) — D-08
```javascript
function estimateSizeMB(durationSec) {
  return durationSec * 44100 * 2 * 2 / 1_000_000;
}
```

### formatSizeMB(mb) — D-08
```javascript
function formatSizeMB(mb) {
  if (mb >= 10) return `~${Math.round(mb)} MB`;
  return `~${mb.toFixed(1)} MB`;
}
```

### stageLabel(status, stage) — from RESEARCH.md Code Examples
```javascript
function stageLabel(status, stage) {
  if (status === 'queued') return 'Na fila...';
  if (status === 'downloading') {
    if (stage === 'checking_duration') return 'Verificando duração...';
    if (stage === 'downloading') return 'Baixando áudio...';
    return 'Baixando...';
  }
  if (status === 'converting') return 'Convertendo para WAV...';
  if (status === 'analyzing') return 'Analisando BPM e tonalidade...';
  return 'Processando...';
}
```

### clearAllTimers() — called at start of submitJob() (Pitfall 4)

function clearAllTimers() {

## Section 7: Event listeners and init

```javascript
function init() {
  // Wire submit button
  $('submit-btn').addEventListener('click', () => {
    const url = $('url-input').value.trim();
    if (!url) return;
    setState('SUBMITTING');
    submitJob(url);
  });

  // Wire retry button (ERROR_JOB state — D-06)
  $('retry-btn').addEventListener('click', () => {
    const url = $('url-input').value.trim();
    if (!url) return;
    setState('SUBMITTING');
    submitJob(url);
  });

  // Enter key on input field triggers submit
  $('url-input').addEventListener('keydown', (e) => {
    if (e.key === 'Enter') $('submit-btn').click();
  });
}

document.addEventListener('DOMContentLoaded', init);
```

IMPORTANT: `submitJob(url)` is an async function. Call it without `await` from the click handler — it is fire-and-forget from the event listener. The function handles its own state transitions internally.
