Script bash com as seguintes seções em ordem:

**Seção 0 — Header e segurança**
- `#!/usr/bin/env bash`
- `set -e` imediatamente após o shebang
- `chmod 750 "$(realpath "$0")"` — auto-chmod obrigatório (Security Gate)
- Verificar que está rodando como root (`[ "$(id -u)" -eq 0 ]`)
- Não adicionar nenhum usuário ao grupo `docker` — grupo `docker` equivale a root; operação futura usa `sudo docker`

**Seção 1 — Preflight (read-only, não modifica nada)**
- `uname -m` — confirmar x86_64 (imprimir valor; se não for x86_64, avisar mas não bloquear — Ubuntu Server 24.04 só instala em amd64 então isso é apenas sanity check)
- `lsb_release -rs` — confirmar 24.04
- `df -h /` — espaço disponível (aviso se < 10GB)
- `free -h` — RAM disponível
- `lscpu` — CPU/modelo/cores para dimensionar concorrência da Phase 13
- `lsblk` — disco HDD/SSD/partições para registrar hardware real
- `swapon --show` — swap atual

**Seção 2 — Prevenção de sleep/hibernate (crítico para notebook servidor)**
- Criar `/etc/systemd/logind.conf.d/nosleep.conf` de forma idempotente:
  ```ini
  [Login]
  HandleLidSwitch=ignore
  HandleLidSwitchExternalPower=ignore
  HandleSuspendKey=ignore
  HandleHibernateKey=ignore
  IdleAction=ignore
  IdleActionSec=0
  ```
- Criar `/etc/systemd/sleep.conf.d/nosleep.conf` de forma idempotente:
  ```ini
  [Sleep]
  AllowSuspend=no
  AllowHibernation=no
  AllowSuspendThenHibernate=no
  AllowHybridSleep=no
  ```
- `systemctl daemon-reload`
- Não reiniciar `systemd-logind` no meio da sessão SSH. Reboot no final aplica a configuração de forma
  previsível; Plan 02 valida o comportamento real com a tampa fechada.

**Seção 3 — Docker via apt repo oficial**
- Instalar dependências: `apt-get update && apt-get install -y ca-certificates curl gnupg lsb-release`
- Remover pacotes conflitantes recomendados pela Docker, tolerando ausência:
  `apt-get remove -y docker.io docker-compose docker-compose-v2 docker-doc podman-docker containerd runc || true`
- Criar `/etc/apt/keyrings/docker.asc` com a chave GPG oficial da Docker
  URL: `https://download.docker.com/linux/ubuntu/gpg`
  Idempotente: verificar se arquivo existe antes de baixar
- Detectar arch/codename com `dpkg --print-architecture` e `. /etc/os-release`
- Validar que `VERSION_CODENAME=noble`; se não for `noble`, abortar com erro claro
- Adicionar repo: `deb [arch=${ARCH} signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu ${VERSION_CODENAME} stable`
  Idempotente: verificar se `/etc/apt/sources.list.d/docker.list` existe antes de criar
- `apt-get update && apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin`
- `systemctl enable --now docker`
- Não executar `usermod -aG docker`. Imprimir aviso: grupo `docker` equivale a root; usar `sudo docker`.

**Seção 4 — Firewall UFW básico**
- Instalar UFW: `apt-get install -y ufw`
- Configurar defaults:
  - `ufw default deny incoming`
  - `ufw default allow outgoing`
- Permitir SSH de forma explícita antes de ativar firewall:
  - `ufw allow 22/tcp comment 'SSH admin access'`
- Se interface `tailscale0` existir: `ufw allow in on tailscale0 comment 'Tailscale private access'`
- Ativar UFW apenas depois das regras de acesso administrativo: `ufw --force enable`
- Imprimir `ufw status verbose` na verificação final
- Nota para fases futuras: UFW não é suficiente para validar portas publicadas por Docker; Phase 15 deve considerar `DOCKER-USER`.

**Seção 5 — Swap (4GB)**
- Idempotente: se `/swapfile` não existir, criar com `fallocate -l 4G /swapfile` e fallback
  `dd if=/dev/zero of=/swapfile bs=1M count=4096 status=progress`
- Garantir `chmod 600 /swapfile`
- Rodar `mkswap /swapfile` apenas quando o arquivo ainda não estiver formatado como swap
- Ativar com `swapon /swapfile` apenas se `/swapfile` ainda não estiver ativo em `swapon --show`
- Persistência em `/etc/fstab` — verificar se já existe entrada antes de adicionar:
  `/swapfile none swap sw 0 0`
- `sysctl vm.swappiness=10` + persistir em `/etc/sysctl.d/99-swappiness.conf`

**Seção 6 — Cgroups v2 (verificação)**
- Ubuntu Server 24.04 usa cgroups v2 por padrão — não precisa modificar nada
- Verificar: `cat /sys/fs/cgroup/cgroup.controllers | grep memory`
- Se não contiver `memory`, imprimir aviso claro (não abortar — raro no Ubuntu 24.04)

**Seção 7 — Watchdog systemd**
- Criar `/etc/systemd/system.conf.d/10-watchdog.conf` de forma idempotente:
  ```ini
  [Manager]
  RuntimeWatchdogSec=15
  ShutdownWatchdogSec=2min
  ```
- `systemctl daemon-reload`
- Nota: o kernel expõe `/dev/watchdog` via módulo iTCO_wdt (Intel) ou sp5100_tco (AMD)
  systemd faz feed automático via /dev/watchdog se disponível; não exigir o módulo aqui
- Watchdog ativo não bloqueia Phase 12 se o hardware não expuser `/dev/watchdog`; Plan 02 documenta.

**Seção 8 — Verificação final**
Imprimir uma tabela de status com todos os componentes:

```

  NOTEBOOK SETUP — VERIFICAÇÃO FINAL
========================================

CPU/RAM/DISCO:  [resumo de lscpu/free/lsblk]
Docker:         [output de docker --version ou ERRO]
Docker run:     [orientar validar com sudo docker run --rm hello-world após reboot]
UFW:            [output de ufw status verbose]
Swap:           [output de swapon --show --noheadings ou NENHUM]
Cgroup Version: [output de docker info grep "Cgroup Version" ou ERRO]
Lid Switch:     [output de systemctl show logind grep HandleLidSwitch]
Watchdog conf:  [cat 10-watchdog.conf RuntimeWatchdogSec line ou AUSENTE]
========================================
PRÓXIMO PASSO: sudo reboot
Após reboot, conectar via Tailscale SSH e reportar outputs ao Claude.
========================================
```

<threat_model>
## Trust Boundaries

| Boundary | Descrição |
|----------|-----------|
| Operador → apt repo Docker | Script baixa chave GPG e pacotes do repositório oficial Docker — HTTPS + verificação de assinatura |
| Script → /etc/systemd/ | Escreve drop-ins como root — idempotente, sem sobrescrever configs existentes |
| Script → /swapfile | Cria arquivo como root com chmod 600 — acesso restrito |
| Moisés → notebook (sudo bash) | Operador executa o script como root — confia no conteúdo commitado no repositório |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Supply Chain | Docker GPG key download via HTTPS | accept | HTTPS + apt verifica assinatura antes de instalar; risco residual pequeno vs convenience script |
| T-12-02 | Privilege Escalation | Docker daemon rootful | mitigate | Não adicionar usuário ao grupo `docker`; operações futuras usam `sudo docker`/systemd root-owned |
| T-12-03 | Tampering | /etc/fstab modificado pelo script | mitigate | Verificar existência da entrada antes de adicionar; `set -e` aborta em caso de falha |
| T-12-04 | DoS | swap insuficiente se HDD falhar durante criação | mitigate | Fallback dd com status progress; falha aborta script via set -e |
| T-12-05 | DoS/Lockout | UFW ativado durante setup remoto | mitigate | Permitir SSH antes de ativar; permitir `tailscale0` quando disponível; Plan 02 valida reconexão pós-reboot |
</threat_model>
