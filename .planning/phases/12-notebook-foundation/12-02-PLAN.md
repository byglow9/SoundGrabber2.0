    PASSO 4 — Após reboot, coletar os outputs de verificação:
    ```bash
    uname -m
    lsb_release -rs
    lscpu | sed -n '1,20p'
    free -h
    lsblk
    docker --version
    sudo docker run --rm hello-world
    sudo docker info 2>&1 | grep "Cgroup Version"
    sudo docker info --format '{{.CgroupVersion}}'
    test -f /sys/fs/cgroup/cgroup.controllers && cat /sys/fs/cgroup/cgroup.controllers
    swapon --show
    grep -n '/swapfile' /etc/fstab
    cat /etc/sysctl.d/99-swappiness.conf
    sudo ufw status verbose
    systemctl show logind | grep HandleLidSwitch
    systemctl show logind | grep -E 'HandleLidSwitch|IdleAction'
    cat /etc/systemd/logind.conf.d/nosleep.conf
    cat /etc/systemd/sleep.conf.d/nosleep.conf
    cat /etc/systemd/system.conf.d/10-watchdog.conf
    systemctl show -p RuntimeWatchdogUSec
    ls -l /dev/watchdog* || true
    tailscale status
    ```

    PASSO 5 — Teste real de tampa fechada:
    ```bash
    date
    ```
    Fechar a tampa por 2-5 minutos. Depois reconectar via Tailscale SSH e rodar:

    ```bash

    loginctl show-seat seat0
    ```

    limitação de hardware, documentar explicitamente que a operação será com tampa aberta ou semiaberta.

## Trust Boundaries

| Boundary | Descrição |
|----------|-----------|
| Moisés → notebook (SSH/Tailscale) | Operador se conecta ao notebook via rede Tailscale privada — conexão autenticada |
| notebook → repositório git | git pull e git push requerem credenciais do Moisés |
| outputs → 12-SETUP-LOG.md | Outputs do sistema copiados para arquivo de log — sem dados sensíveis esperados |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-08 | Information Disclosure | 12-SETUP-LOG.md commitado no repo | accept | Log contém apenas outputs de comandos de sistema — sem senhas, tokens ou dados pessoais |
| T-12-09 | Tampering | git pull puxando código modificado antes da execução | accept | Moisés verifica o conteúdo do script antes de executar com sudo — padrão de operação normal |
| T-12-10 | Denial of Service | Reboot interrompe sessão SSH ativa | accept | Reboot é intencional e esperado; operador sabe que vai reconectar — instrução explícita no passo 3 |
| T-12-11 | Denial of Service | UFW bloqueia acesso remoto | mitigate | Plan 01 permite SSH/Tailscale antes de ativar; Plan 02 valida reconexão pós-reboot e `ufw status verbose` |
