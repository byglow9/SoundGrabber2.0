<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API `/featured` -> visitor DOM | Operator-entered content crosses from trusted backend storage into public browser rendering. |
| Featured link -> third-party site | Visitor leaves SoundGrabber to external URL in a new browser tab. |
| Public HTML -> route discovery | Public page could accidentally reveal the hidden operator route. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-07 | Elevation of Privilege | `static/app.js` featured rendering | mitigate | Use `textContent` for all operator-provided text and avoid `innerHTML` for featured content. |
| T-11-08 | Spoofing | `.featured-link` anchors | mitigate | Set `target="_blank"` and `rel="noopener"` on every external link. |
| T-11-09 | Information Disclosure | `static/index.html` | mitigate | Keep `/yonkou` absent from public HTML, buttons, links, menus, and copy. |
| T-11-10 | Denial of Service | `loadFeatured()` | mitigate | Fetch once on `DOMContentLoaded`, silently ignore failures, and never block the downloader workflow. |
| T-11-11 | Tampering | CSS/layout constraints | mitigate | Preserve table layout and static CSS tests so the sidebar cannot introduce modern layout regressions or always-reserved empty space. |
</threat_model>
