<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator browser -> `/yonkou/login` | Secret password enters API process. |
| Operator browser -> signed cookie | Session token returns to browser and is replayed on protected updates. |
| Operator browser -> `POST /featured` | Curated text and external URLs become public content. |
| API -> Redis/fallback JSON | Current release is persisted and loaded for public visitors. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Elevation of Privilege | `_require_admin`, `POST /featured` | mitigate | Validate signed timed cookie with `itsdangerous`, require it on every operator write, and return 401 on missing/invalid cookie. |
| T-11-02 | Spoofing | `POST /yonkou/login` | mitigate | Compare password with constant-time comparison against `ADMIN_PASSWORD`; use 5/min rate limit. |
| T-11-03 | Tampering | `FeaturedReleaseRequest` | mitigate | Enforce Pydantic validators for non-empty fields, max three links, and http/https URLs before storage. |
| T-11-04 | Denial of Service | New routes | mitigate | Add slowapi limits: 60/min read routes, 10/min operator write, 5/min login. |
| T-11-05 | Information Disclosure | Auth error responses | mitigate | Use generic `Admin login required` / `Senha incorreta` style responses; do not expose signing errors or configured password state. |
| T-11-06 | Denial of Service | Redis outage | mitigate | Catch Redis connection/timeout exceptions and use configured JSON fallback path. |
</threat_model>
