<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Visitor browser -> GET /featured | Public unauthenticated read endpoint returns operator-entered content. |
| Operator browser -> POST /yonkou/login | Password crosses from browser to API. |
| Operator browser -> POST /featured | Authenticated operator content crosses into storage and later public rendering. |
| API -> Redis/fallback JSON | Curated release persists in Redis and fallback file. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Elevation of Privilege | `/yonkou/login`, session cookie | mitigate | Wave 0 tests require constant-time password check behavior, signed HttpOnly SameSite cookie, and 5/min login rate limit. |
| T-11-02 | Tampering | `POST /featured` | mitigate | Wave 0 tests require valid operator session, Pydantic payload validation, max three links, and http/https URL validation. |
| T-11-03 | Denial of Service | `/featured`, `/yonkou/login`, `/featured` POST | mitigate | Wave 0 tests require slowapi rate limits of 60/min, 5/min, and 10/min respectively. |
| T-11-04 | Information Disclosure | Public homepage | mitigate | Wave 0 tests require no visible `/yonkou` link/copy/affordance in public HTML. |
| T-11-05 | Elevation of Privilege | Public featured rendering | mitigate | Wave 0 tests require `textContent`, no featured `innerHTML`, and `rel="noopener"` on external links. |
| T-11-06 | Denial of Service | Redis outage | mitigate | Wave 0 tests require graceful JSON fallback for `featured:current`. |
</threat_model>
