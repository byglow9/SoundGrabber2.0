
DECISÃO DE DESIGN (resolvida em revisão de plano — checker blocker):
O probe verifica apenas "o servidor está respondendo?" — NÃO "o servidor retorna 2xx?".
Qualquer resposta HTTP (200, 404, 500) significa que o bgutil está up e aceitando conexões.
Somente `httpx.RequestError` (ConnectError, ConnectTimeout, etc.) indica que o servidor está
inacessível. Sem verificação de `resp.is_success` — isso eliminaria falsos negativos no caso
de bgutil retornar 404 na rota raiz `/` quando saudável.

De 10-RESEARCH.md §Padrão 2 — except em tasks.py:
```python
# Inserir ANTES de "except RuntimeError as e:" (linha 104):
```

De 10-RESEARCH.md §Pitfall 5 — ordem dos excepts:
```
BgutilUnavailable é subclasse de RuntimeError.
Python processa excepts em ordem — o primeiro match vence.
Se except RuntimeError vier ANTES de except BgutilUnavailable,
a exceção de bgutil é silenciosamente capturada pelo handler genérico
e produz error_type="download_error" (ERRADO).
OBRIGATÓRIO: except BgutilUnavailable ANTES de except RuntimeError.
```

<threat_model>
## Trust Boundaries

| Boundary | Descrição |
|----------|-----------|
| pipeline.py → bgutil HTTP | probe HTTP para URL controlada por operador via env var BGUTIL_BASE_URL |
| pipeline.py → yt-dlp | chamada após probe confirmar disponibilidade |
| api/tasks.py → pipeline.py | captura de exceções tipadas |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Spoofing | bgutil probe endpoint | accept | BGUTIL_BASE_URL é env var configurada pelo operador, não input de usuário. Sem vetor de spoofing externo. |
| T-10-02-02 | Information Disclosure | BgutilUnavailable mensagem expõe BGUTIL_BASE_URL | accept | A URL é exposta intencionalmente per D-07: "facilita debug do operador". Não é dado sensível — é endereço de rede privado Railway. |
| T-10-02-03 | Denial of Service | probe httpx.get timeout 2s | mitigate | Timeout de 2s limita bloqueio do worker. Job falha rápido em vez de travar indefinidamente esperando bgutil. |
| T-10-02-04 | Elevation of Privilege | except BgutilUnavailable antes de RuntimeError | accept | A ordem evita que RuntimeError genérico absorva silenciosamente falhas de infra. Comportamento correto por design. |

- `os.chmod(wav_path, 0o600)` em download_audio permanece inalterado (SEC-FILE-01)
- Rate limiting nos endpoints existentes inalterado
- Nenhum novo endpoint HTTP introduzido — Security Gate não é acionado para novos controles
