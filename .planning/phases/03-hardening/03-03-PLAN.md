--- MUDANCA 3: Registrar limiter no app e handler 429 ---

Inserir logo apos a linha `app = FastAPI(...)`, ANTES do handler `_validation_exception_handler`:

```python
app.state.limiter = limiter


def _rate_limit_handler(request: Request, exc: RateLimitExceeded) -> JSONResponse:
    """Formato unificado {error, error_type} com Retry-After header (D-04).

    _inject_headers adiciona Retry-After, X-RateLimit-Limit, X-RateLimit-Remaining.
    headers_enabled=True no Limiter e necessario para _inject_headers funcionar.
    """
    response = JSONResponse(
        status_code=429,
        content={
            "error": "Too many requests. Try again in a moment.",
            "error_type": "rate_limit_error",
        },
    )
    response = request.app.state.limiter._inject_headers(
        response, request.state.view_rate_limit
    )
    return response


app.add_exception_handler(RateLimitExceeded, _rate_limit_handler)
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cliente HTTP -> slowapi Limiter | IP extraido por get_ipaddr; X-Forwarded-For pode ser forjado por cliente malicioso |
| slowapi Limiter -> Redis | Contadores de rate limit armazenados no Redis compartilhado entre workers |
| POST /jobs handler -> Celery | Apos passar o rate limit, job e enfileirado no Celery |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-dos-flood | Denial of Service | POST /jobs | mitigate | slowapi 3/min por IP com Redis backend compartilhado entre todos os workers Uvicorn; contador nao pode ser bypassado trocando de worker |
| T-info-disclosure | Information Disclosure | _rate_limit_handler | mitigate | Body 429 e {error, error_type} sem internals do slowapi; nao expoe limite exato (apenas "Try again in a moment") |
| T-dos-disk | Denial of Service (disco) | sweep_expired_wavs | mitigate | Implementado em Plan 02; .part e .ytdl deletados em <= 20 minutos |
| T-ip-spoof | Elevation of Privilege | get_ipaddr / X-Forwarded-For | accept | get_ipaddr le X-Forwarded-For se presente; cliente pode forjar o header em ambiente sem ProxyHeadersMiddleware. Mitigacao completa (uvicorn --proxy-headers + trusted proxy) adiada para v1.1 per STATE.md |
