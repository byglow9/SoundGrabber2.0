## Spacing Scale

Esta fase usa espaçamento em px bruto via atributos HTML (`cellpadding`, `cellspacing`) e `style` inline. Sem CSS variables, sem classes utilitárias.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Espaçamento interno de células, gap entre label e valor |
| sm | 8px | Padding de células de resultado, gap entre linhas da tabela |
| md | 16px | Padding do input, padding do botão submit |
| lg | 24px | Separação entre seções da tabela externa |
| xl | 32px | Margem superior/inferior da tabela outer container |

Exceptions: Touch targets não se aplicam (desktop-only, fixed 640px). Atributos `cellpadding` e `cellspacing` são preferidos sobre CSS quando possível. Inline `style` apenas onde atributos HTML são insuficientes.

## Typography

| Role | Font | Size | Weight | Line Height | Smoothing |
|------|------|------|--------|-------------|-----------|
| Site title (`#site-title`) | Dela Gothic One | 48px | Regular (400 — único peso disponível) | 1.1 | disabled |
| Button text (`#submit-btn`, `#download-link`) | Dela Gothic One | 16px | Regular | 1.0 | disabled |
| BPM / Key values primários (`#bpm-value`, `#key-value`) | Sligoil | 32px | Regular (400) | 1.2 | disabled |
| Tagline (`#site-tagline`) | Sligoil | 14px | Regular | 1.4 | disabled |
| Body / UI text geral | Sligoil | 13px | Regular | 1.4 | disabled |
| Labels uppercase (`BPM`, `TONALIDADE`, etc.) | Sligoil | 11px | Regular — `text-transform: uppercase` | 1.3 | disabled |

### Font smoothing — regra global obrigatória

```css
body {
  -webkit-font-smoothing: none;
  -moz-osx-font-smoothing: grayscale;
  text-rendering: optimizeSpeed;
}
```

### Font loading

```css
@font-face {
  font-family: 'Dela Gothic One';
  src: url('/static/fonts/DelaGothicOne-Regular.woff2') format('woff2');
  font-display: block;
}

@font-face {
  font-family: 'Sligoil';
  src: url('/static/fonts/Sligoil-Micro.woff2') format('woff2');
  font-display: block;
}
```

`font-display: block` — texto permanece invisível até a fonte carregar. Sem FOUT.

## Color

| Role | Hex | Usage |
|------|-----|-------|
| Dominant — fundo (60%) | `#000000` | Page background, input background, retry button background no estado default |
| Accent — texto + borda (40%) | `#ff8800` | TODO: ver nota abaixo |
| Accent — hover escurecido | `#ff6600` | Hover de submit, hover de download, focus do input |
| Accent — placeholder | `#804400` | Placeholder do `#url-input` (50% de #ff8800 em hex puro, sem rgba) |
| Destructive — erro de borda | `#ff3300` | Border do `#url-input` no estado ERROR_VALIDATION (aplicado via JS inline style) |

**Nota de distribuição:** A paleta é monocromática radical — `#ff8800` atua tanto como cor de texto quanto de borda e fundo de botão primário. Não há cor secundária de 30%. A distribuição é: 60% preto (#000000) de fundo + 40% laranja-âmbar (#ff8800) para todo o resto. Isso é autêntico à referência NFO/warez (BeatMasters International 2000).

### Elementos reservados para cada cor

**`#ff8800` é usado exclusivamente em:**
- Todo texto da página (títulos, labels, valores, body)
- Todas as bordas de elementos (`<table>`, input, retry button borda)
- Background do botão submit (`#submit-btn`)
- Background do botão download (`#download-link`)

**`#000000` é usado em:**
- Background da página (`body`)
- Background do input (`#url-input`)
- Texto do botão submit e download (inverted: bg laranja + texto preto)
- Background do retry button no estado default

**`#ff6600` é usado em:**
- Hover de `#submit-btn` (`:hover`)
- Hover de `#download-link` (`:hover`)
- Focus de `#url-input` (`:focus` border-color)

**`#804400` é usado exclusivamente em:**
- Placeholder de `#url-input`

**`#ff3300` é usado exclusivamente em:**
- Border de `#url-input` no estado ERROR_VALIDATION (via JS, não CSS)

## Layout Contract

### Regras estruturais (não-negociáveis)

- Layout construído com `<table>` — zero `display: flex`, zero `display: grid`
- Zero CSS custom properties (`--qualquer-coisa`) em style.css
- Largura fixa: 640px centrada via `align="center"` no outer table (atributo HTML, não CSS)
- `box-sizing: border-box` NÃO usado — content-box padrão (autenticidade Y2K)
- `border-radius: 0` — sem bordas arredondadas
- `box-shadow: none` — sem sombras
- `transition: none`, `animation: none`, `transform: none` — sem propriedades CSS modernas
- `cursor: pointer` nos botões (era suportado no IE6 — permitido)

### Hierarquia de tabelas

```
<table width="640" align="center">         ← outer container
  <tr><td id="header">                     ← título + tagline
  <tr><td id="form-area">                  ← input + botão
  <tr><td id="duration-hint">              ← aviso 15 minutos
  <tr><td id="progress-area">              ← label de progresso (hidden por default)
  <tr><td id="result-card">                ← card de resultado (hidden por default)
  <tr><td id="error-area">                 ← área de erro (hidden por default)
```

### Inner table — result card (2 colunas)

```
<table width="100%" id="result-card">
  <tr>
    <td width="50%" id="result-bpm">      ← BPM principal + metade + dobro
    <td width="50%" id="result-key">      ← Tonalidade + Camelot
  <tr>
    <td colspan="2" id="result-size">     ← tamanho estimado + download (largura total)
```

### IDs DOM obrigatórios (27 total — JS não pode quebrar)

Todos os IDs do Phase 4 devem sobreviver à conversão div→table intactos. O JS referencia apenas pelo ID — o tipo de elemento (`<td>`, `<tr>`, `<table>`) é irrelevante.

```
app, header, site-title, site-tagline, form-area, duration-hint,
input-group, url-input, submit-btn, validation-error, progress-area,
progress-label, result-card, result-bpm, result-key, result-size,
download-area, bpm-value, bpm-half-value, bpm-double-value,
key-value, camelot-value, size-value, download-link, error-area,
error-message, retry-btn
```
