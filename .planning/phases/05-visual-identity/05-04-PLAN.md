**ATENÇÃO CRÍTICA — result-card hidden:**
O HTML acima usa `<td hidden>` no TR pai do result-card, e `id="result-card"` na inner table.
O app.js faz `$('result-card').hidden = true/false` — portanto o elemento com `id="result-card"` deve aceitar o atributo `hidden`. Uma `<table>` aceita `hidden` normalmente.

Porém isso pode causar comportamento inesperado: o app.js seta `.hidden` no elemento `#result-card` (a inner table), mas o TR pai (`<td hidden>`) também precisa ser controlado.

**Solução correta:** Mover o `id="result-card"` e o `hidden` inicial para o `<tr>` ou `<td>` de container, e colocar a inner table sem ID:

```html
    <!-- Result card row (DONE state) -->
    <tr>
      <td id="result-card" hidden>
        <table width="100%" cellpadding="8" cellspacing="0">
          <!-- ... conteúdo interno igual ... -->
        </table>
      </td>
    </tr>
```

Desta forma, `$('result-card').hidden = false` exibe o `<td>` que contém toda a inner table. Isso está alinhado com o DOM Analysis do RESEARCH.md: "result-card: elemento alvo = `<table>` inner (2 cols)" — mas o JS só precisa que o elemento com id="result-card" responda a `.hidden`. Usar `<td id="result-card">` é mais simples e correto.

**Escrever o HTML FINAL com esta correção aplicada** — `<td id="result-card" hidden>` como container, inner table sem ID.
