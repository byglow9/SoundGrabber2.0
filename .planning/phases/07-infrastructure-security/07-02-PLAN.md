<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet -> Railway edge -> uvicorn | Railway termina TLS no edge; uvicorn recebe HTTP do proxy interno; HSTS instrui browsers a sempre usar HTTPS na proxima visita |
| Application startup -> Redis | Aplicacao verifica forma da REDIS_URL antes de aceitar trafego; se sem senha em producao, falha imediatamente |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7-01 | Elevation of Privilege | Redis broker | mitigate | `_check_redis_auth(settings.redis_url, settings.dev_mode)` no lifespan: levanta RuntimeError se URL nao contem `@` e DEV_MODE nao esta ativo. Producao Railway sempre tem senha (D-08); DEV_MODE so existe em local (D-14). |
| T-7-04 | Tampering (downgrade HTTPS->HTTP, MITM) | Cliente browser | mitigate | Header `Strict-Transport-Security: max-age=31536000; includeSubDomains` injetado em todas as respostas via _security_headers middleware. Apos primeira visita HTTPS, browser refuta acessos HTTP por 1 ano. |
| T-7-LOCAL-HSTS | Spoofing (HSTS pin em localhost) | Browser local do desenvolvedor | accept | Risco baixo: HSTS pin em localhost afeta apenas localhost, nao outros dominios. RFC 6797 sec 7.2 recomenda que browsers ignorem HSTS recebido via HTTP. Se ocorrer, dev limpa HSTS no browser settings. Documentado em RESEARCH.md Pitfall 4. |
| T-7-CONFIG-LEAK | Information Disclosure | Mensagem de erro do RuntimeError | accept | A mensagem nao vaza nem REDIS_URL completo nem credenciais — apenas instrui o operador a definir credenciais. Sem PII, sem segredos. |
</threat_model>
