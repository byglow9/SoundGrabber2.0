    4. `healthcheckPath = "/health"` — esta rota ja existe em api/main.py linha 339 (SEC-API-03 da Phase 6) e retorna 200 quando Redis esta saudavel. Railway usa este endpoint para decidir se o deploy esta pronto para receber trafego.
    5. `healthcheckTimeout = 60` — 60 segundos eh suficiente para o lifespan rodar `_check_redis_auth` + iniciar sweeper thread + responder o primeiro health check.
    6. `restartPolicyType = "ON_FAILURE"` com `restartPolicyMaxRetries = 3` — se _check_redis_auth levantar RuntimeError (Redis sem senha em producao por configuracao errada), Railway tenta 3 reinicios e depois marca o deploy como falho. Isso evita loop infinito de restart consumindo creditos de plano.
